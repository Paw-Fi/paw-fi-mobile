# Task Completion Checklist

When a task is completed:
1.  **Verify**: Ensure the code changes meet the user's requirements.
2.  **Lint**: Run `flutter analyze` on the modified files to ensure no new issues were introduced.
3.  **Test**: If applicable, run relevant tests (`flutter test`) or manually verify standard behavior.
4.  **UI Review**: If UI changes were made, verify them against design goals (e.g., "Apple-like", dark mode support).
5.  **Documentation**: Update relevant memories or documentation if architectural changes occurred.
6.  **Report**: Summarize the changes and any verification steps taken to the user.
